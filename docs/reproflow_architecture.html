ReproFlow 给小白看的架构说明

让 AI 写模型，让框架管实验

ReproFlow 的目标很直接：你把数据和论文放进固定位置，AI 按规范实现模型，框架负责数据读取、训练、指标、baseline、随机种子和结果输出。

适合谁 会用 AI 写代码，但还不熟悉深度学习项目组织的人。
解决什么 避免每篇论文都变成一堆散乱脚本。
默认输出 一个 log、一个 history CSV、一个 best checkpoint。

先记住四步

1

放数据

把 CSV 放到 dataset/，再用 configs/data 说明哪些列是特征、哪个列是 label。

2

放论文

把 PDF 或 Markdown 放到 docs/papers/，AI 后续从这里理解要复现的方法。

3

接模型

模型放在 models/，参数放在 configs/model，输出统一为 {"logits": logits}。

4

跑对比

用同一份数据、同一组指标和同一批随机种子对比 baseline、论文方法和消融版本。

项目里每一层负责什么

你主要会改
  • dataset/
    放自己的数据
  • configs/data/
    说明数据列和任务类型
  • docs/papers/
    放要复现的论文
  • configs/experiment/
    组织完整对比实验
AI 主要会改
  • models/
    实现新模型或论文方法
  • configs/model/
    写模型超参数
  • paper_methods/
    记录论文方法和复现说明
  • configs/tuning/ / configs/ablation/
    补调参和消融配置
框架帮你管
  • Data_pre.py
    读取、切分、预处理数据
  • Dataset.py
    统一 batch 格式
  • engine.py
    训练循环和 checkpoint
  • metrics/
    统一计算评估指标

第一次训练只需要这三条命令

先检查数据、模型和 trainer 能否对上：

python scripts/doctor.py data=sample_binary model=transformer trainer=binary metrics=default

再跑一个最小训练：

python main.py data=sample_binary model=transformer trainer=binary metrics=default training_loop.epochs=1
小白最常见的问题集中在数据列名、label、task_type、trainer 对齐上。先跑 doctor.py，可以提前发现这类问题。

默认会生成什么

训练日志 training_*.log

记录每轮训练和验证指标，方便看训练过程。

指标表格 history_*.csv

每一行是一轮 epoch，适合后续画图和对比。

最佳模型 checkpoints/*.pth

保存当前 monitor metric 最好的模型权重。

AI 复现论文时看哪里

先让 AI 读这些文件：

AGENTS.md
docs/ai_reproduction_guide.md
.claude/skills/reproflow-reproduce-paper/SKILL.md
paper_methods/template/

然后生成论文方法模板：

python scripts/paper_methods/scaffold.py paper_xxx \
  --paper docs/papers/paper_xxx.pdf \
  --dataset my_task \
  --trainer binary

什么时候再看高级功能

想调参数
  • 看 configs/tuning/
  • 运行 scripts/tuning/run_grid_search.py
想做消融
  • 看 configs/ablation/
  • 运行 scripts/ablation/run_ablation.py
想公平对比
  • 看 configs/experiment/
  • 运行 scripts/experiment/run_experiment.py
ReproFlow keeps the first run simple: data config, model config, trainer, metrics, log, CSV, checkpoint.
